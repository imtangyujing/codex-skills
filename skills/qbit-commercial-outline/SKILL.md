# 科技媒体商稿提纲 Skill

为量子位风格的科技自媒体商业稿件撰写结构化提纲。

---

## 核心认知

商稿提纲不是简单的信息罗列,是一篇文章的骨架设计。好的提纲要同时解决三个问题:
1. 读者凭什么点进来?(标题+开头的钩子)
2. 读者凭什么看完?(叙事节奏和信息密度)
3. 读者看完会记住什么?(核心结论和价值判断)

---

## 提纲结构模板

### 一、标题方向(2-5个备选)

**原则**
- 必须有信息增量,不能是空洞的形容词堆砌
- 优先使用具体数字、场景细节、反差感
- 语气偏口语化、自媒体化,带点态度,不要新闻联播腔
- 可以用比喻、类比、网络梗增加记忆点

**常见模式**
- 数据冲击型: 1张A10扛40路并发,这个AI老师跑进了百万学生家里
- 反差/悬念型: 昔日商业之王SaaS,已死
- 身份标签型: 央企第一家AI独角兽!
- 场景画面型: 亦庄马拉松的小孩哥机器人,这次在春晚讲小品
- 痛点共鸣型: 跨境电商的自动驾驶来了

### 二、开头(占全文10%-25%)

**核心任务**: 用最短的篇幅制造好奇心缺口,让读者必须往下看。

**常用钩子类型**

1. 案例冲击: 先扔一个具体的、反直觉的案例或数据,再揭示背后主角
   - 适用: 产品已有可量化的落地成果(遨虾: 转化率暴涨5倍、与爱为舞: 百万学生)
   
2. 悬念反转: 抛一个行业共识或疑问,再用主角打破它
   - 适用: 公司有沉寂期或外界存在认知偏差(它石智航: 拿完钱消失一年,然后呢?)

3. 判断先行: 直接下一个大胆判断,用身份/事件背书
   - 适用: 公司有里程碑事件(中电信AI: 央企第一家AI独角兽,诞生了)

4. 痛点共鸣: 从行业痛点切入,描述大家都知道但没解决的问题,引出破局者
   - 适用: 产品解决的问题足够普遍(百融云创: SaaS已死; 斑马口语: 妈妈再也不用担心)

5. 借势入场: 借一个正在发酵的行业热点事件开场,先讲热点,再引出「但这条路线并非XX首创」,将主角定位为先行者/同路人
   - 适用: 有强力第三方验证、行业热点可借势的场景(原力灵机: 借GEN-1热度引出DM0的具身原生路线)
   - 关键: 热点是引子不是主角,开头要完成从热点到客户的过渡,并且让客户的位置比热点更靠前(先行者/更早交卷)

**开头结构公式**
钩子(1-3句) → 主角亮相(1句) → 核心成绩/事件(1-2句) → 设问引出正文(1句)

**开头的叙事节奏偏好**
- 开头不要一股脑把信息堆完,要有层次递进感。先给一个让读者驻足的点(数字/判断/热点),再一步步加码
- 每一步之间可以插入短句做呼吸: 「是的,不是XX,也不是XX,而是......」「信号非常明确。」
- 行业定性判断放在主角亮相之前,先让读者理解这件事为什么重要,再揭示谁做到了
- 开头最后一句可以用设问收束(「具身原生,到底有什么魔力?」),也可以用一句极简的重锤事实句收束(「从中科大少年班到硅谷市值千亿美元的上市公司CTO,葛小川只用了三年」)

**第一句定调的重要性**
- 商稿开头的第一句不是铺陈背景,是情绪钩子。要让读者在零点几秒内决定继续读下去
- 通稿式开场是大忌(「美东时间X月X日,XX公司发布公告」),这种写法属于官方新闻稿,不属于自媒体商稿
- 好的第一句往往带态度、带判断、甚至带网络感: 「华人真撑起硅谷半边天了……」「SaaS,已死。」「这一次,真的不一样了。」
- 可以用省略号、感叹号、口语化表达,不要怕「不正式」

**公司/产品的定位话术要前置**
- 读者第一次听到一家公司时,必须在开头就拿到「这家公司有多厉害」的关键信息
- 用横向对标的方式快速定位(「仅次于Google和Meta的第三大移动广告巨头」比「领先的广告技术公司」有效十倍)
- 用一个冲击性数字佐证(「光去年一年股价就暴涨700%」),数字要具体、要短、要可感
- 这一层信息最好在开头前三分之一就交付完

**引用CEO/主角原话是加分项**
- 如果有CEO公告、主角采访金句,可以用引用块(>)的形式嵌入开头,增加可信度和画面感
- 引用不要长,一两句最精华的即可,可以用省略号裁剪
- 引用之后要接自己的判断或过渡句,不能让引用孤零零地结束

### 三、正文(分2-3个大段,每段占20%-40%)

**三段式是主流结构,每段承担不同叙事功能:**

**模式A: 产品力→技术力→行业意义(最常用)**
- 第一段(What): 产品做了什么? 解决了什么痛点? 效果如何?
- 第二段(How): 怎么做到的? 技术架构、数据、训练方法
- 第三段(Why): 为什么重要? 行业格局、未来展望、价值升华

适用: 大多数产品/技术类商稿(与爱为舞、斑马口语、遨虾)

**模式B: 事件→实力→意义**
- 第一段: 里程碑事件本身(融资、发布、合作)
- 第二段: 凭什么? 产品、技术、团队、数据壁垒
- 第三段: 这意味着什么? 对行业/生态的影响

适用: 以事件驱动的稿件(中电信AI: 融资; 它石智航: 新产品发布)

**模式C: 幕后→合作→趋势**
- 第一段: 揭秘/幕后故事(增加可读性)
- 第二段: 合作方各自的角色和技术贡献
- 第三段: 行业趋势判断

适用: 联合商稿、事件报道(火山x松延: 春晚幕后)

**模式D: 产品力→商业模式→生态**
- 第一段: 产品/解决方案全貌
- 第二段: 商业模式创新(定价、交付、考核)
- 第三段: 生态布局和壁垒

适用: 企业服务/B端商稿(百融云创: 产品→RaaS→硅碳共治)

**模式E: 行业验证→客户实力→趋势定性**
- 第一段: 行业热点/第三方验证事件,拆解其核心判断,顺势引出客户走的是同一条路
- 第二段: 客户的产品/技术详细展开,证明客户不仅同路,而且更早/更强
- 第三段: 拉高到行业趋势层面,给出定性判断(XX元年/XX时刻)

适用: 有强力第三方背书、需要「借他人之口验证客户路线」的稿件(原力灵机: GEN-1验证→DM0详解→具身原生元年)

注意: 模式E的第一段不是在写别人,是在用别人的判断为客户的路线铺路。第三方的篇幅要克制,核心论点讲清楚即可,不要喧宾夺主。

**模式F: 公司势能→人物反差→群体升华**
- 第一段: 主角所在公司有多热门/多值钱,用数据和对标快速定位
- 第二段: 人物本身的故事,重点是「非典型」反差感和关键选择
- 第三段: 把个案拉高到群体层面,给出趋势判断或时代定性

适用: 人事任命类商稿、人物聚焦类稿件(葛小川: AppLovin势能→葛小川其人→华人在硅谷新篇章)

注意: 模式F的第二段是全稿核心,要有「人味儿」,不能写成履历简介。人物段的信息排序应该是「反差特质→性格金句→关键选择/豪赌→成就→教育背景」,而不是「教育背景→履历→成就→性格」这种通稿式顺序。教育背景和工作履历放在最后做补充,效果远好于开场亮明身份。

### 四、每段内部的写法

**段落开头**: 用一句话概括本段核心论点或制造过渡

**正文展开遵循: 背景→痛点→解法→结果 的循环**
- 背景: 行业现状/客户处境(1-2句)
- 痛点: 具体的、可感知的问题(不要抽象,要场景化)
- 解法: 主角的产品/技术如何解决(技术细节适度展开)
- 结果: 数据、案例、对比(有数字就上数字)

**子话题的展开方式**
- 每个子话题下只列关键要点,不要堆论据。论据本身不重要,重要的是论点和推理逻辑,用一个最关键的论据把意思说清楚即可
- bullet数量控制: 每个子话题下3个要点是正常的,4个已经偏多,超过4个必须砍
- 不要在bullet前写「论点:XXX」这种显式标注,直接把论点作为子话题标题本身
- 缩进层级: 数字列表(1. 2. 3.)和短横线(-)是两种不同视觉层级,短横线本身已经构成视觉下降,前面不要再加tab或空格缩进

**子话题的密度与节奏**
- 每个子话题下的要点不要平均用力。标注哪个子话题是本段的「比例重点」,提示正文撰写时需要详细展开
- 子话题之间要有叙事衔接,不能只是并列罗列。用过渡句串联: 「不过,GEN-1可能并非首个具身原生模型」「智能,正是如此从物理交互中原生长出来」
- 最后一个子话题尽量是一句收束性判断,为进入下一大段做铺垫

**技术段落特别注意**
- 不要堆砌术语,每个技术点都要回答so what
- 用类比让非专业读者也能感受到厉害之处
- 可以用不可能三角、哥德巴赫猜想等比喻增加记忆点
- 技术指标先给结论再给数字: 「参数量不是壁垒,原生设计才是。」然后才列2.4B、60ms等数据

**人物段落特别注意**
- 人物段的核心任务是让读者「记住这个人」,不是罗列履历
- 信息排序很关键: 反差特质 → 性格金句 → 关键选择/豪赌 → 成就 → 教育背景。教育背景和工作履历放最后,做补充说明,不要开场就亮
- 善用人物原话: 如果有采访金句或公开演讲原文,用引用块(>)嵌入,比转述有力得多。引用块可以单独成行,不用包在bullet里
- 性格细节比title更重要: 「骑摩托车上高速上班」比「全球首席产品及工程官」更能让读者记住这个人
- 人物的「非典型」标签要在段落前两个子话题内立起来,后面的履历和成就才不会变成通稿
- 关键节点(如加入某公司、做某选择)可以用「背景/原因/成果」三段式展开,比平铺直叙更有叙事感

**借势/对比类段落特别注意**
- 当段落涉及第三方(竞品/行业事件),要明确第三方的角色: 是「验证者」「对照组」还是「合作方」
- 第三方的核心论点提炼为2-3个bullet,不要展开太多细节
- 每讲完第三方的一个论点,立刻衔接客户的对应优势,不要等第三方全部讲完再切客户

### 五、收尾

**常见收法**
- 价值升华: 从个案回到行业,点明这件事对更多人的意义(斑马口语→教育公平; 与爱为舞→因材施教)
- 先发优势: 强调马太效应,暗示窗口期正在关闭(百融云创、与爱为舞)
- 愿景落地: 用一个具体的未来场景收束,而非空洞口号(火山x松延: 消费级机器人带出去玩)
- 定性命名: 给当下时刻一个标签/命名,用判断句收束(2026年→具身原生元年)。适合趋势定调类稿件
- 群体升华: 从一个个案/人物,拉升到所属群体的时代意义(葛小川→华人在硅谷新篇章)。适合人物类/事件类稿件

**收尾的节奏偏好**
- 可以借用第三方/行业大佬的一句话作为跳板,再落到自己的判断上
- 最后2-3句要有力度,倾向于短句+判断句收束,不要长句拖尾
- 收尾可以包含一句具象化的场景(让机器人进入工厂,代替人类完成危险辛苦的工作),让抽象判断落地

**收尾段的输出形式**
- 升华/定性为主的收尾,用**逐字稿形式**输出,而不是提纲式bullet。因为收尾的情绪节奏和句式才是价值所在,用bullet无法传达
- 逐字稿收尾的典型特征: 每句话单独成行、大量短句、可以用破折号强化语气(「这不是XX一个人的故事。这是属于XX的——XX新篇章。」)
- 不要在收尾段用数字列表罗列「核心判断」,那样像PPT总结页,不像收尾
- 如果收尾涉及具体案例/数据,再考虑用子话题结构展开

## 风格偏好

### 语气
- 自媒体口语化,不是新华社通稿
- 适度使用网络语言和流行梗(doge、小孩哥、春卷、卷)
- 可以带主观判断和态度,但不油腻
- 允许出现量子位第一人称视角(量子位注意到、量子位经过一通深挖)

### 节奏
- 短句为主,长句偶尔穿插
- 单独成段的短句用于强调或转折(然后就没了声音。)
- 设问句推动阅读(究竟是怎么做到的? 凭什么?)
- 每段之间有明确的逻辑过渡,不要硬跳

### 禁区
- 不用破折号（——）
- 不用引号（""''「」）
- 中英文之间不加空格
- 所有标点符号使用中文版本：冒号用：不用:,括号用（）不用(),分号用；不用;
- 不要假大空的套话（赋能、助力、引领,能不用就不用）
- 不要通篇吹捧,要有信息密度

### 比例标注
- 每个段落标注占全文的百分比,总和100%
- 开头一般10%-25%,视钩子需要的展开程度而定
- 正文各段根据信息量灵活分配

## 输出格式

输出**Markdown格式**,以便于客户直接复制到飞书等文档工具使用。全文所有标点符号使用中文标点(包括冒号:、括号()、逗号,、句号。、百分号%、等号=等),引号统一使用「」,不要使用""''。

**标题方向:** 用二级或三级标题(`## 标题方向`),下方用数字列表列出备选标题

**开头(X%):** 用逐字稿形式输出,即写出接近成稿的完整文字,而非概括性描述。因为开头的节奏感和叙事风格是提纲中最需要让客户提前感知的部分,用梗概无法传达这种感觉。开头中可以嵌入CEO公告/主角原话引用,用Markdown的`>`引用块呈现

**正文第一段:段落主题(X%):** 用二级或三级标题写段落主题,下方用数字列表写子话题,每个子话题下用短横线列出要点

后续正文段落同理。

**收尾段:** 如果收尾以升华/定性为主,用**逐字稿形式**输出(参考开头的写法),每句单独成行,**不要用数字列表**。如果收尾有较多具体内容(如案例、数据),才用子话题+要点展开。

**格式细节:**
- 数字列表(1. 2. 3.)用于子话题层级,短横线(-)用于要点层级,两者是不同的视觉层级
- 短横线(-)本身已经是视觉下降,前面**不要**加tab或空格缩进
- 子话题下的要点数量控制: 3个正常,4个偏多,超过4个必须砍
- 不要用「论点:XXX」这种显式前缀,直接把论点作为子话题标题本身

**其他要求:**
- 提纲写到子话题+要点级别,不需要写成完整句子(开头和收尾除外)
- 论据本身不重要,重要的是论点。每个要点只放一个最能说明问题的论据即可,不要罗列多个同质论据
- 关键数据、案例、技术名词在提纲阶段就要写清楚
- 如果有待确认/待采访补充的内容,用(待补充)(待采访确认)标注

## 灵活适配

以上是方法论框架,不是死板模板。实际撰写时根据稿件素材灵活调整:

- 素材以采访对话为主 → 正文可以大比例采用对话原文(如它石智航70%对话)
- 联合商稿/双方合作 → 需要平衡双方露出(如火山x松延)
- 偏B端/商业模式 → 加重商业逻辑论述(如百融云创)
- 偏C端/消费产品 → 加重场景描写和用户故事(如斑马口语、遨虾)
- 央企/国资背景 → 适度体现责任担当叙事,但不要变成宣传稿(如中电信AI)
- 借势/路线验证类 → 第一段用第三方热点铺路,第二段才是客户主场,比例上客户段要明显重于第三方段(如原力灵机: GEN-1段35% vs DM0段35%,但DM0段的信息密度和展开深度要远大于GEN-1段)
