# Workflow

## Stage Map

1. `待建稿`
   - 输入:分工文档、嘉宾资料表、负责人规则、目标文件夹。
   - 动作:`assignment_doc_builder.py`校验嘉宾姓名、title、主题、负责人，只创建总稿骨架，开放可编辑权限。
   - 输出状态:`已建总稿骨架`、`待骨架确认`。

2. `待骨架确认`
   - 输入:总稿URL和总稿骨架内容。
   - 动作:人工检查总稿骨架是否符合模板、嘉宾名单、title、主题、圆桌结构和空bullet要求。
   - 输出状态:`已骨架确认`。

3. `已骨架确认`
   - 输入:已确认的总稿骨架、目标文件夹、嘉宾资料表。
   - 动作:`assignment_doc_builder.py`创建单人空文档和独立圆桌文档，单人文档只放对应section骨架。
   - 输出状态:`已建单人空文档`、`待妙记`。
   - 状态表:`base_status_sync.py`同步Feishu Base，写入总控、单人稿、圆桌稿和圆桌嘉宾跟踪行。

4. `待妙记`
   - 输入:状态表里的录音编号、嘉宾名、公司名、主题、speaker mapping。
   - 动作:`minutes_monitor.py`按命名规范查找飞书妙记。默认每天中午12点、下午6点各检查一次。
   - 输出状态:`已发现妙记`或继续`待妙记`。

5. `已发现妙记`
   - 输入:妙记token或URL。
   - 动作:`minutes_monitor.py`拉取逐字稿、AI总结、章节，保存到指定工作目录。
   - 输出状态:`已拉取逐字稿`。

6. `已拉取逐字稿`
   - 输入:逐字稿、speaker mapping、总稿URL、目标section key。
   - 动作:`viewpoint_writer.py`提炼五条观点，定位总稿section，写回`观点精华提炼`。
   - 输出状态:`已写入总稿`、`待人工确认`。

7. `待人工确认`
   - 输入:总稿section和人工确认标记。
   - 动作:人工修改或确认。脚本只读取确认信号。
   - 输出状态:`已人工确认`。

8. `已人工确认`
   - 输入:总稿section、单人文档URL、圆桌文档URL。
   - 动作:`confirmed_doc_sync.py`把已确认内容同步到对应单人文档或圆桌文档。
   - 输出状态:`已同步单人文档`。

9. `已同步单人文档`
   - 输入:已同步文档列表、命名规则、导出目录。
   - 动作:`word_exporter.py`导出Word。
   - 输出状态:`已导出Word`。

10. `已导出Word`
   - 输入:Word文件路径、Feishu链接、负责人分组。
   - 动作:汇总交付清单。
   - 输出状态:`已交付`。

## Recovery Rules

- 每个脚本先读状态表，只处理自己负责的状态切片。
- 每个脚本最后同步Feishu Base状态表，记录`status`、`updated_at`、`output_url`、`error`、`retry_hint`。
- 失败后保留已生成链接，不重复创建同名文档。优先复用状态表里的URL和doc token。
- 如果Feishu接口限流，串行重试并写入当前进度。
