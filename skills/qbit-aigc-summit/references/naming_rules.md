# 妙记命名规范

## 核心原则

链接只作辅助定位，自动化触发要靠稳定命名。每条妙记标题都应能让脚本直接读出录音编号、嘉宾名、会议场景和内容类型。

## 推荐格式

```text
【录音13】马孝武-AIGC峰会-单人分享
【录音14】庄明浩-AIGC峰会-圆桌讨论
【录音15】冯雷-AIGC峰会-圆桌讨论
```

## 必填信息

- `【录音N】`：使用中文方括号和阿拉伯数字，例如`【录音13】`。编号是主键，后续状态表、目标清单和文档章节都按这个字段对齐。
- `嘉宾名`：紧跟录音编号，使用常用中文姓名。多人圆桌可以写主嘉宾，其他嘉宾放入关键词清单。
- `会议标识`：固定写`AIGC峰会`或本次项目指定简称。
- `内容类型`：建议使用`单人分享`、`圆桌讨论`、`采访`、`补录`等稳定词。

## 可选信息

- 主题短语可以放在末尾，例如`世界模型`、`多智能体`、`AI应用`。
- 如果一条录音拆成多段，在编号后追加小写字母，例如`【录音13a】`、`【录音13b】`。
- 如果嘉宾名需要消歧，在目标清单的`keywords`里补公司名、职位或主题词。

## 脚本匹配口径

- 优先匹配`minute_token`。已知token时可以绕过搜索，直接拉取逐字稿和AI产物。
- 其次匹配录音编号。标题包含`【录音13】`或`录音13`时权重最高。
- 再匹配嘉宾名。标题命中嘉宾名的权重高于元数据命中。
- 最后匹配关键词。关键词用于补强召回，不适合作为唯一主键。
- 链接会保存在状态文件里，方便人工打开复核；自动化判断以命名字段和token为准。

## 检查节奏

- 默认在每天`12:00`和`18:00`后的时间窗口内运行。
- 建议用系统定时任务或外层主控skill触发`scripts/minutes_monitor.py`。
- 如果要临时检查，使用`--force`跳过时间窗口。

## 目标清单格式

```json
{
  "targets": [
    {
      "id": "recording-13-ma-xiaowu",
      "recording_no": "13",
      "guest": "马孝武",
      "keywords": ["AIGC峰会","单人分享"]
    },
    {
      "id": "roundtable-zhuang-minghao",
      "recording_no": "14",
      "guest": "庄明浩",
      "keywords": ["圆桌讨论","世界模型"]
    }
  ]
}
```

## 推荐命令

```bash
python3 scripts/minutes_monitor.py \
  --targets data/minutes_targets.json \
  --status data/minutes_status.json \
  --output-dir data/minutes_artifacts \
  --start 2026-05-21 \
  --end 2026-05-21 \
  --search-scope owned
```

```bash
python3 scripts/minutes_monitor.py \
  --recording-no 13 \
  --guest 马孝武 \
  --keyword AIGC峰会 \
  --force \
  --dry-run
```

## 状态字段

- `not_found`：当前时间范围内没有命名命中的妙记。
- `recording_processing`：妙记已出现，逐字稿或AI产物仍在生成。
- `fetched`：已经调用`lark-cli vc +notes --minute-tokens`并记录产物摘要。
- `dry_run_planned`：只输出计划命令，没有访问飞书，也没有写状态文件。
- `error`：搜索或拉取失败。脚本会把错误消息写入目标状态，便于下次接着排查。

## 维护提醒

- 新增录音时先补目标清单，再确认妙记标题符合`【录音N】嘉宾名-AIGC峰会-内容类型`。
- 同一录音不要频繁改标题；必要修改时同步更新目标清单。
- 圆桌类内容容易多人名混在一起，保留统一录音编号，关键词里补其他嘉宾名。
- 状态表里的链接用于快速打开核验；命名规范负责让自动化稳定运行。
