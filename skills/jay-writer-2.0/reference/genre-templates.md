# 题材模板库

本文件归纳Jay公众号文章各题材的专项写法规范。SKILL.md负责核心风格DNA（语感、节奏、禁区、自检），这里负责每种题材的结构骨架、开头节奏、正文编织方式、结尾升华路径和AI常踩的雷。

写作流程：先读SKILL.md确认风格底线 → 判断题材 → 来这里查对应模板 → 写 → 回SKILL.md跑四层自检。

---

## 笔记型

**已校准范例：** Gowers数学实验、Thariq力推HTML（MD要凉）、Nathan Lambert中国AI行

### 定义

基于某位大佬的长文、博客、报告或访谈，整理成更适合自媒体传播的版本。

核心心态：我看到了一篇有意思的东西，想分享给大家，这是我整理后更方便阅读的版本。

Jay在笔记型里的身份是策展人+评论员。原文作者是主角，Jay是旁白。

### 结构模板

```
【钩子】原文最炸的一句话/判断/事实
  ↓
【身份与语境】原作者是谁、为什么重要（1-3句）
  ↓
【社交证据】大佬反应、评论区共鸣、Jay个人背书
  ↓
【签名过渡】"以下是整理后的版本。Enjoy。"
  ↓
【要点摘要（长文可选）】原文核心发现的bullet list预览
  ↓
【正文板块1】转述 + Jay评论
【正文板块2】转述 + Jay评论
【正文板块N】转述 + Jay评论
（板块之间可穿插社区反馈/质疑声音）
  ↓
【升华】超越原文的更大命题连接
  ↓
【强收束】意象 / 行动呼吁 / 回环呼应
```

### 开头：三板斧 + 签名过渡

笔记型开头有固定节奏，三步走完后用签名过渡句收束进入正文。

**第一步：钩子。** 原文里最抓人的一句话、一个判断、一个事实，直接甩出来。不铺垫，先把读者注意力抓住。可以是原作者金句（blockquote），也可以是Jay对原文核心观点的一句话提炼。

范例（Gowers篇）：
> 如果AI的数学水平继续按目前速度发展，我们（数学研究者）很快会面临一场危机。

范例（MD篇）：
> Markdown，当死。

**第二步：这人是谁、为什么重要。** 1-3句交代原作者身份和文章语境。"聊天时顺口提一嘴"的方式，不是百科式介绍。

范例：
> 这是艾伦研究所（Ai2）的研究员Nathan Lambert，在最近结束中国之行后，发自内心的一番感慨。

**第三步：社交证据。** 这篇文章在圈内引发了什么反响？谁跟帖了？评论区炸了没有？解决"凭什么我要花时间读这个"的问题。如果没有明确社交证据，用Jay自己的判断来背书。

范例（Jay个人背书）：
> 我是真的觉得，这种一手的、主观的信息，才是这个时代最重要的东西。

范例（大佬背书）：
> 连卡帕西都表示说的太对，反手给出一个认可。

范例（社区反馈）：
> 其实吧，如果光Thariq这么说，不能代表什么。关键评论区共鸣的人太多了，这本身就代表一种趋势。

三步走完，用签名过渡收束：

**"以下是整理后的，更方便大家阅读的版本。Enjoy。"**

这句话（或近似变体）是笔记型的仪式性过渡，标志从"为什么要读"进入"开始读"。每篇笔记型文章都应出现。

**可选加分动作：**

- **要点摘要预览**：信息量特别大的原文（如Nathan中国行），可在签名过渡之前，用bullet list把最关键的几个发现先列出来，给读者心理地图。每条要足够具体、有画面感，像预告片高光镜头。
- **双声部开头**：原文涉及两个以上重要人物不同视角时（如Gowers+陶哲轩），在开头把两个声部都亮出来，制造张力。"两位当世最顶尖的数学家，面对同一场风暴，给出了不同角度的思考。"

### 正文：转述—评论交替编织

笔记型正文最核心的技巧是"转述—评论"交替编织。每转述一个板块，就插入一段Jay的声音。不是先把原文全部转述完再集中点评。

**转述段规则：**

- 用自己的话重述原文核心信息，关键判断和金句用blockquote保留原味
- 遇到需要科普的概念，用自己的类比通俗化。类比要有画面感和原创性（"就像搭积木""就像逼着一个画家用筷子画油画"），不用俗套比喻
- 原文有具体数据、时间线、案例，保留并强化——这些是信息密度的来源
- 忠实呈现原作者观点，不偷换、不裁剪成只对你论点有利的部分

**评论段规则：**

- 这一段是Jay的声音。几种姿态：
  - 共鸣："这一点我是真的有共鸣"
  - 延伸："说到这个，其实xxx"
  - 质疑："但这里有个问题"
  - 连接自己经历："我们自己内部也有一个类似的xxx"
- 评论段要短、有温度、像插话。不能变成长篇大论盖过原文
- Jay的评论总篇幅不超过原文转述篇幅。大约7:3（转述:评论）。原文是主菜，Jay是调料

**社区反馈段（推荐纳入）：**

- 原文引发了有意思的讨论或质疑，主动纳入。引用网友评论、大佬跟帖、反面观点
- 尤其当原文观点有争议时，纳入pushback会增强可信度
- 范例（MD篇纳入token成本质疑）：网友算了一笔账，同样内容换HTML后成本涨了近一倍。甚至有人调侃这是Anthropic更快榨干usage的新办法

**减法原则：**

- 笔记型原文通常很长，必须做减法。优先保留：最反直觉的判断、最有画面感的细节、最能引发共鸣的段落
- 删掉的内容不需要交代。但不能删掉对原文结论至关重要的逻辑环节

### 结尾：超越原文的升华

笔记型结尾不能停在"以上就是原文内容"。必须有Jay自己的升华，从原文"自然生长出来"。

**三条升华路径（从范例提炼）：**

1. **具体→时代命题**：原文聊具体领域 → Jay连接到更大的时代问题。范例：格式之争 → 哥白尼式智能观，"我们正在经历一场认知上的哥白尼革命"
2. **谨慎→多推一步**：原文结论谨慎 → Jay给出更有力的意象。范例：陶哲轩的搅拌机比喻 → "不要沦为黑客帝国里，被插着管子的生物电池"
3. **隐含→挑明**：原文有一个没说透的meta观点 → Jay把它挑明。范例：Nathan观察到的建设文化 → "这跟中国的开源精神，显然是一脉相承的"

结尾要有强收束。笔记型的收束往往比原文更有力度——Jay在转述过程中积累了势能，结尾是释放的时刻。

### 自检专项（配合SKILL.md L3-5使用）

笔记型文章的L3-5检查清单：

- [ ] 开头三板斧完整？（钩子→身份→社交证据→签名过渡）
- [ ] 原文信息忠实还原？没有歪曲或遗漏关键观点？
- [ ] "转述—评论"交替编织？（而非通篇转述无Jay声音）
- [ ] 评论篇幅克制？（不超过转述）
- [ ] 有没有纳入社区反馈/反面声音？
- [ ] 结尾有超越原文的升华？
- [ ] 通俗化类比有画面感和原创性？

### AI写笔记型的常见踩雷点

1. **太"总结报告"**：通篇平铺直叙转述，没有Jay的评论穿插。读起来像摘抄，没有活人在跟你聊
2. **开头缺社交证据**：直接进内容转述，没有建立"为什么这值得读"
3. **评论段太弱或太强**：要么完全没Jay的声音，要么喧宾夺主。正确比例7:3
4. **结尾停在原文层面**：原文说到哪结尾就到哪，没有Jay自己的升华
5. **遗漏反面声音**：只呈现正面信息，没纳入pushback或质疑
6. **通俗化类比不够原创**：用"打个比方说"引出平庸比喻。Jay的类比要有画面感、有意外感（"积木""筷子画油画""性缩力"）
7. **签名过渡句缺失**：没有"以下是整理后的版本。Enjoy。"，开头和正文之间缺仪式感
